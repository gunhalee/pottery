# Consepot 보류 범위 및 코드 흔적 보관 문서

작성일: 2026-05-05  
목적: 이번 구현 범위에서 제외하거나 보류한 항목을 삭제하지 않고, 추후 재개 가능한 상태로 보관하기 위한 기록

---

## 1. 현재 구현 범위에 대한 결정

이번 단계의 기준은 다음과 같다.

- 브랜드명은 `콩새와 도자기공방`으로 둔다.
- 상품 흐름은 `Consepot 내부 상세 + 카페24 구매 CTA`로 간다.
- 결제는 Consepot 자체 결제창이 아니라 카페24 주문서에서 처리한다.
- Toss는 직접 API 연동이 아니라 `카페24 결제수단/PG`로 포함한다.
- 도메인은 새로 구매하고, 추후 메인 도메인은 Vercel, `shop.` 서브도메인은 카페24 쪽으로 연결하는 방향을 검토한다.
- 카카오채널 URL과 네이버예약 URL은 아직 확정되지 않았으므로 환경변수/placeholder로 준비한다.
- slug 매핑은 초기에 `slug -> cafe24ProductNo/cafe24ProductUrl` 수동 매핑으로 시작하고, 상품 수가 늘면 카페24 API 동기화로 확장한다.

## 2. 보류하되 삭제하지 않을 항목

아래 항목들은 이번 범위의 우선 구현 대상은 아니지만, 나중에 다시 켤 수 있도록 코드 흔적을 보관한다.

| 항목 | 현재 판단 | 보관 이유 |
|---|---|---|
| Consepot 자체 Toss 결제창/결제위젯 | 보류 | 지금은 카페24 주문서와 PG 설정을 기준으로 하되, 향후 자체 주문서가 필요할 수 있음 |
| Toss 결제 승인 API | 보류 | 자체 결제를 선택할 때 `paymentKey`, `orderId`, `amount` 검증 흐름으로 재사용 가능 |
| Toss webhook | 보류 | 자체 결제 또는 비동기 결제수단을 붙일 때 결제 상태 동기화에 필요 |
| 자체 주문 DB/주문 관리자 | 보류 | 카페24가 주문 기준 시스템이지만, 향후 CRM/운영 분석이 필요해질 수 있음 |
| 커스텀 오더 | 초기 제외 | 현재 정책상 제외지만, 향후 상담형 주문이나 제작 의뢰가 열릴 수 있음 |
| 이메일 뉴스레터 | 초기 제외 | 고객 알림은 카카오채널 중심이나, 장기적으로 이메일 전환 가능성 있음 |
| 로그인 기반 커뮤니티/마이페이지 | 초기 제외 | 수강생 페이지, 멤버십, 구매 이력 확장 가능성 있음 |
| 인스타그램 피드 자동 연동 | 초기 제외 | 갤러리/뉴스 콘텐츠 자동화 시 재검토 가능 |
| 파일 업로드/이미지 관리 | 초기 제외 | 운영자 CMS 또는 작품 이미지 관리가 필요해질 수 있음 |
| 자체 예약 draft | 보류 | 클래스 예약은 네이버예약 중심이나, 내부 예약 문의/대기자 관리로 전환 가능 |
| 텀블벅 직접 연동 | 초기 제외 | 프로젝트 링크 섹션으로 시작하고, 직접 연동은 추후 검토 |
| 스마트스토어/네이버 광고/네이버 블로그 자동 연동 | 초기 제외 | 운영 채널 확장 단계에서 별도 검토 |

## 3. 현재 저장소에 남아 있는 관련 흔적

아래 파일/섹션은 삭제하지 않는다. 구현 중 혼동되지 않도록 보류 상태임을 알고 접근한다.

### Toss 직접 결제 관련

- `src/app/api/toss/confirm/route.ts`
  - 현재 501 스텁이다.
  - 이번 범위에서는 사용하지 않는다.
  - 향후 Consepot 자체 결제를 선택할 때 결제 승인 API로 확장한다.

- `src/app/api/webhooks/toss/route.ts`
  - 현재 501 스텁이다.
  - 이번 범위에서는 사용하지 않는다.
  - 향후 Toss webhook 상태 동기화에 사용한다.

- `src/app/(site)/success/page.tsx`
- `src/app/(site)/fail/page.tsx`
  - 현재 결제 성공/실패 placeholder 페이지다.
  - 카페24 주문서 redirect만 사용할 경우 필수는 아니다.
  - 향후 자체 결제 또는 카페24 완료 후 재유입 페이지로 재정의 가능하다.

### 커스텀 오더 관련

- `src/app/actions/custom-order.ts`
  - 현재 미구현 서버 액션이다.
  - 이번 범위에서는 호출하지 않는다.
  - 커스텀 오더 정책이 열릴 때 상담/견적 액션으로 확장한다.

- `src/app/(site)/shop/page.tsx`의 `Custom Order` 섹션
- `src/components/layout/site-footer.tsx`의 `/shop#custom` 링크
- `src/app/globals.css`의 `.custom-box`, `.custom-action`
  - 이번 MVP에서 노출 여부는 별도 UX 판단이 필요하다.
  - 삭제하지 않고, 보류 또는 숨김 처리 후보로 둔다.

### 뉴스레터/알림 관련

- `src/app/actions/subscribe.ts`
  - 현재 미구현 서버 액션이다.
  - 이번 범위에서는 이메일 뉴스레터를 만들지 않는다.
  - 카카오채널 중심 운영 후 이메일 전환이 필요하면 확장한다.

- `src/lib/notifications/index.ts`
  - 현재 비어 있다.
  - 카카오채널 링크 이상의 알림 시스템을 만들 때 사용한다.

### 예약/클래스 관련

- `src/app/actions/reservation.ts`
  - 현재 미구현 서버 액션이다.
  - 이번 범위에서는 네이버예약 CTA를 우선한다.
  - 내부 예약 문의/대기자 기능이 필요할 때 확장한다.

### 인스타그램/업로드/CMS 관련

- `src/app/api/instagram/route.ts`
  - 현재 501 스텁이다.
  - 이번 범위에서는 인스타그램 자동 피드를 붙이지 않는다.

- `src/app/api/uploads/sign/route.ts`
  - 현재 501 스텁이다.
  - 이번 범위에서는 파일 업로드나 운영자 CMS를 만들지 않는다.

- `src/db/schema/index.ts`
- `src/lib/db/index.ts`
  - 현재 비어 있다.
  - 자체 DB가 필요해질 때 사용한다.

## 4. 구현 시 주의사항

- 보류 항목 파일은 삭제하지 않는다.
- 사용하지 않는 스텁은 새 MVP 흐름에서 import하거나 호출하지 않는다.
- 사용자가 요청하기 전까지 Toss 직접 결제 API 구현은 진행하지 않는다.
- Toss는 카페24 PG 설정 범위로 문서화하고, Consepot 코드에서는 구매 CTA와 안내 문구만 담당한다.
- 커스텀 오더는 정책상 초기 제외이므로 새 CTA나 폼을 추가하지 않는다.
- 클래스 예약은 네이버예약 URL 확정 전까지 CTA fallback을 둔다.
- 카카오채널/네이버예약/카페24 도메인은 환경변수 기반으로 준비한다.

---

## 5. 재개 조건

아래 조건이 충족되면 보류 항목을 다시 검토한다.

- 자체 결제가 필요해지고, 카페24 주문서가 브랜드 경험 또는 운영 요구를 충족하지 못하는 경우
- 카페24 재고/주문 기준 시스템과 별도 주문 DB 사이의 동기화 책임을 감당할 수 있는 경우
- 커스텀 오더 운영 정책, 제작 가능 범위, 견적 기준이 확정된 경우
- 이메일 수신 동의, 발송 도구, 개인정보 처리방침이 준비된 경우
- 수강생 전용 페이지나 멤버십 운영 정책이 확정된 경우
- 콘텐츠 운영자가 인스타그램, 이미지 업로드, CMS 자동화를 실제로 필요로 하는 경우
