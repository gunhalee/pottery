## 1. 프로젝트 개요

### 1.1 배경 및 목적

도자기 공방의 수익 증대를 위한 디지털 전략의 핵심 허브로서 웹사이트를 구축한다. 인스타그램·아이디어스·숨고 등 외부 플랫폼을 통한 마케팅 활동의 모든 종착지가 되며, 방문자를 팬으로 전환하는 관계 맺기 플랫폼이다.

> 핵심 원칙: 방문자가 낯선 사람으로 들어와서, 팬으로 나가는 흐름을 설계한다.

### 1.2 전략적 3축

| 만들기 | 알리기 | 팔기 |
|---|---|---|
| 공방 스토리·도자기 철학으로 브랜드 신뢰 형성 | SNS 콘텐츠·갤러리로 감성 노출 확산 | 제품·클래스·B2B로 실제 수익 전환 |

### 1.3 수익 구조 목표

| 수익 채널 | 수단 | 목표 비중 |
|---|---|---|
| 제품 판매 | 아이디어스·스마트스토어·홈페이지 | 40% |
| 클래스 수익 | 원데이·정기·단체 클래스 | 35% |
| B2B 납품 | 카페·기업 답례품 | 15% |
| 주문제작 | 커스텀 각인·맞춤 제작 | 10% |

## 2. 글로벌 네비게이션

> 네비게이션바는 모든 페이지에 고정 노출되며, 방문자가 어느 페이지에서든 소개·소식·참여·소장으로 즉시 진입할 수 있어야 한다.

### 2.1 구성 항목

| 항목 | 링크 | 비고 |
|---|---|---|
| 로고 / 공방명 | / (홈) | 클릭 시 홈으로 이동 |
| 소개 | /intro | 주인장·도자기·공방 |
| 소식 | /news | 작업 일지·일정 |
| 참여 | /class | 클래스 소개·예약 |
| 소장 | /shop | 작품 구매·주문제작 |

### 2.2 동작 요구사항

- **Sticky 고정**: 스크롤 시에도 화면 상단에 항상 고정 노출
- **현재 페이지 활성화 표시**: 현재 위치한 페이지의 메뉴 항목 하이라이트
- **모바일 햄버거 메뉴**: 모바일에서는 햄버거 아이콘 → 슬라이드 다운 or 풀스크린 메뉴
- **로고 클릭 시** 홈(/) 이동
- **카카오채널 바로가기 아이콘**을 네비게이션 우측에 고정 노출

## 3. 페이지 구조 및 정보 아키텍처

### 3.1 사이트맵

> 방문자 흥미도 레벨: 스침(홈) → 궁금함(소개) → 관심(소식) → 욕구(참여·소장)

| 페이지 | 목적 | 흥미도 레벨 | 렌더링 전략 |
|---|---|---|---|
| 홈 (/) | 3초 안에 끌리게 | Lv.1 스침 | SSG + ISR |
| 소개 (/intro) | 궁금함을 신뢰로 | Lv.2 궁금함 | SSG |
| 소식 (/news) | 관심을 연결로 | Lv.3 관심 | SSG + ISR |
| 참여 (/class) | 욕구를 경험으로 | Lv.4 욕구 | SSR |
| 소장 (/shop) | 애착을 소유로 | Lv.4 욕구 | SSG + ISR |

## 4. 페이지별 상세 기능 요구사항

### 4.1 홈 (/)

| 항목 | 내용 |
|---|---|
| 목적 | 첫 방문자에게 3초 안에 브랜드를 각인시키고, 각 페이지로 자연스럽게 유도 |
| 핵심 전략 | 감성 비주얼 → CTA 3개 → 인스타 피드 → 신뢰 지표 |

**섹션 구성**

- **히어로 섹션**
  - 자동재생 배경 영상 또는 감성 이미지 슬라이드
  - 브랜드 슬로건 한 줄 텍스트 오버레이
  - 소개·소식·참여·소장 진입 CTA 버튼 4개
- **제작 과정 미리보기**
  - 흙 → 성형 → 굽기 → 완성 타임랩스 영상 임베드 (YouTube/Vimeo)
- **갤러리 하이라이트**
  - 최신 작품 6~8점 그리드 노출, 소장 페이지 링크
- **인스타그램 피드**
  - Instagram Basic Display API 자동 연동, 최신 9개 노출
  - Vercel Edge Cache로 캐싱 (30분 주기 갱신)
- **신뢰 지표**
  - 수강생 수·작품 판매 수·클래스 누적 횟수 숫자 카운터

---

### 4.2 소개 (/intro)

| 항목 | 내용 |
|---|---|
| 목적 | 궁금함을 신뢰로 전환. 사람 → 도자기 → 공방 순서로 감성적 연결 |
| 핵심 전략 | 작가의 사람됨이 먼저, 공간은 나중 |

**섹션 구성**

- **주인장 소개**
  - 작가 사진 + 작업 철학 텍스트
  - 핵심 키워드: 왜 도자기인가, 어떤 마음으로 만드는가
- **도자기 이야기**
  - 제작 과정 사진 갤러리 (흙의 질감, 유약, 가마)
  - 각 작품에 담긴 의미와 철학
- **공방 소개**
  - 공방 외관·내부 사진
  - 카카오맵 or 네이버지도 API 연동 (주소·오시는 길)
  - 운영 시간·주차 정보

---

### 4.3 소식 (/news)

| 항목 | 내용 |
|---|---|
| 목적 | 관심을 연결로. 공방이 살아있음을 느끼게 하고, 이어가기로 자연 유도 |
| 핵심 전략 | 소식을 읽는 사람은 이미 팔로우할 준비가 된 사람 |

**섹션 구성**

- **작업 일지 / 블로그**
  - Notion CMS 연동 — 주인장이 노션에 작성 → 웹 자동 반영 (ISR 10분)
  - 태그 필터: 작업일지·일정·비하인드·신제품
- **일정 캘린더**
  - 클래스 일정·플리마켓 참가·전시 일정 표시
  - Google Calendar API 연동 or Notion DB 기반
- **이어가기 내장 (소식 하단)**
  - 인스타그램 팔로우 버튼
  - 카카오채널 친구추가 버튼
  - 이메일 구독 폼 → Stibee API 연동

---

### 4.4 참여 (/class)

| 항목 | 내용 |
|---|---|
| 목적 | 욕구를 경험으로 전환. 클래스 탐색 → 일정 확인 → 예약·결제까지 원스톱 |
| 핵심 전략 | 진입장벽을 최소화한 예약 플로우 |

**클래스 유형**

| 유형 | 설명 | 예약 방식 |
|---|---|---|
| 원데이 클래스 | 1회성, 누구나 참여 가능, 최고 수익성 | 네이버 예약 연동 or 자체 예약폼 |
| 정기 클래스 | 월 4회, 고정 수입 확보 | 자체 예약폼 + 결제 |
| 단체·기업 클래스 | 팀빌딩, B2B 제안용 | 문의 폼으로 연결 |

**핵심 기능 요구사항**

- **클래스 상세 카드**
  - 클래스명·유형·소요시간·가격·정원·잔여석 표시
  - 수강생 후기 노출 (관리자 승인 구조)
- **스케줄 캘린더**
  - 월별 클래스 일정 캘린더 뷰
  - 실시간 잔여석 표시 (SSR 처리)
  - 잔여석 0 시 자동 품절 처리
- **예약 및 결제**
  - 네이버 예약 iframe 연동 (초기)
  - 자체 예약폼 + 토스페이먼츠 SDK (성장 단계)
  - 예약 완료 시 카카오 알림톡 발송

### 4.5 소장 (/shop)

| 항목 | 내용 |
|---|---|
| 목적 | 애착을 소유로 전환. 작품 탐색 → 구매 또는 주문제작 연결 |
| 핵심 전략 | 단순 쇼핑이 아닌 작품을 들이는 경험으로 설계 |

**상품 라인업**

| 라인 | 가격대 | 예시 품목 | 구매 방식 |
|---|---|---|---|
| 보급형 | 1~3만원 | 소품·컵 | 아이디어스·스마트스토어 링크 |
| 중가 | 5~10만원 | 식기류·세트 | 아이디어스·스마트스토어 링크 |
| 프리미엄 | 15만원 이상 | 작품급 단품 | 홈페이지 직접 문의 |
| 세트·각인 | 별도 협의 | 커스텀 주문제작 | 주문제작 폼 |

**핵심 기능 요구사항**

- **작품 갤러리**
  - 카테고리 필터: 보급형·중가·프리미엄·세트·각인
  - 재고 상태 뱃지: 판매중·한정수량·품절
  - Notion DB 연동, 작품 추가 시 ISR 자동 갱신
- **구매 연결**
  - 아이디어스·스마트스토어 외부 링크 포워딩
  - 자체 결제 도입 시 토스페이먼츠·네이버페이·카카오페이 연동
- **주문제작 신청 폼**
  - 입력 항목: 용도·수량·예산·참고 이미지 첨부·희망 일정
  - 제출 시 → 이메일(Resend) + 카카오 알림톡 동시 발송

## 5. 기술 요구사항

### 5.1 기술 스택

| 영역 | 기술 | 비고 |
|---|---|---|
| 프레임워크 | Next.js 14 (App Router) | SSG·SSR·ISR 혼합 사용 |
| 배포 | Vercel | 자동 배포, Edge Network, 무료 플랜 시작 |
| 스타일링 | Tailwind CSS | 모바일 우선 반응형 |
| CMS | Notion API (초기) → Sanity (성장) | 주인장 직접 콘텐츠 관리 |
| 결제 | 토스페이먼츠 SDK | 네이버페이·카카오페이·카드 지원 |
| 이메일 | Resend | 문의 알림·구독 확인 메일 |
| 구독 CRM | Stibee | 뉴스레터, 무료 1,000명까지 |
| 분석 | GA4 + Vercel Analytics + MS Clarity | 방문자·전환·히트맵 |

### 5.3 외부 API 연동

| 서비스 | 용도 | 적용 페이지 | 우선순위 |
|---|---|---|---|
| Instagram Basic Display API | 릴스·피드 자동 연동 | 홈 | **필수** |
| 카카오맵 or 네이버지도 API | 공방 위치 | 소개 | **필수** |
| 카카오채널 | 채팅 문의 버튼 | 전체 | **필수** |
| Notion API | 소식·작품·클래스 CMS | 소식·참여·소장 | **필수** |
| 네이버 예약 | 클래스 예약 | 참여 | **필수** |
| Stibee API | 이메일 구독 | 소식 | 권장 |
| 토스페이먼츠 | 자체 결제 | 참여·소장 | 성장 단계 |
| Google Calendar API | 클래스 일정 실시간 연동 | 소식·참여 | 선택 |

- 기본 SEO: 페이지별 타이틀·메타디스크립션·OG 태그 설정
- 네이버 서치어드바이저 등록 필수
- Google Search Console 등록
- 구조화 데이터: 공방 위치·클래스 일정 검색 노출 최적화
- 소식 블로그: "도자기 선물 추천", "원데이 클래스 후기" 등 키워드 SEO

## 7. 분석 및 시장 파악

> 홈페이지 자체가 가장 정확한 시장 조사 도구가 된다. 방문자 행동 데이터가 곧 시장 반응이다.

| 지표 | 의미 | 도구 |
|---|---|---|
| 작품별 클릭률 | 어떤 라인에 관심 많은지 | GA4 이벤트 |
| 클래스 예약 클릭 | 어떤 클래스 수요 높은지 | GA4 전환 |
| 구매 링크 클릭 | 어느 플랫폼 전환율 높은지 | UTM + GA4 |
| 스크롤 깊이 | 어느 콘텐츠에서 이탈하는지 | MS Clarity |
| 유입 채널 | 인스타·아이디어스 중 효과적인 채널 | UTM 파라미터 |
