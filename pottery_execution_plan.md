# 도자기 공방 홈페이지 개발 실행계획서

- 문서 버전: v1.0
- 작성일: 2026-04-24
- 상태: Draft
- 기준 문서
  - `pottery_prd.md`
  - `pottery_technical_spec.md`

## 1. 문서 목적

이 문서는 홈페이지 구축을 실제 개발 순서에 맞게 쪼갠 실행용 계획서다.
기술명세서가 "무엇을 만들 것인가"를 정리한 문서라면, 이 문서는 "어떤 순서로 만들 것인가"를 정리한다.

이 문서는 아래 용도로 사용한다.

- 개발 착수 전 작업 순서 정렬
- 주차별/단계별 완료 기준 확인
- 병렬 작업 가능 범위 파악
- 오픈 직전 누락 항목 점검

## 2. 전체 개발 순서 요약

```text
0. 프로젝트 기초 결정 정리
1. 저장소/앱 스캐폴딩
2. 공통 레이아웃 및 디자인 시스템
3. 콘텐츠 모델 설계 및 Notion 연동
4. 정적 페이지 구현 (홈/소개)
5. 콘텐츠 페이지 구현 (소식/소장)
6. 트랜잭션 DB 및 예약 도메인 구축
7. 참여 페이지 + 예약/결제 구현
8. 문의/구독/알림 연동
9. 분석/SEO/성능 최적화
10. QA/배포/운영 전환
```

## 3. 단계별 실행 계획

## 3.1 Phase 0. 착수 준비

### 목표

- 구현에 필요한 외부 서비스 계정과 키 발급 준비
- 개발 전 반드시 확정되어야 하는 운영 기준 정리

### 작업

- 도메인, 공방명, 대표 연락처, 대표 이메일 최종 확인
- Vercel 프로젝트 생성
- Supabase 프로젝트 생성
- Notion integration 생성 및 DB 공유
- Toss Payments 테스트 상점 생성
- Kakao Developers 앱 생성 및 도메인 등록
- Resend 도메인/발신자 설정
- 이미지/영상/카피 등 브랜딩 자산 폴더 정리

### 산출물

- 환경변수 목록 초안
- 외부 서비스 접근 권한 정리 문서
- 브랜드 자산 폴더

### 선행조건

- 없음

### 완료 기준

- 개발자가 필요한 계정과 키를 모두 받을 수 있다
- 테스트 환경에서 각 외부 서비스 접속 가능 여부를 확인했다

개발 중 일정 압박이 생기면 아래 순서로 우선순위를 지킨다.

1. 예약/결제/문의 등 전환 기능을 먼저 완성한다.
2. 홈/소개/참여/소장 핵심 흐름을 먼저 완성한다.
3. 인스타그램 자동 연동과 카카오 알림톡 실발송은 후순위로 미룰 수 있다.
4. 관리자 페이지는 v1 범위 밖이므로 추가하지 않는다.
