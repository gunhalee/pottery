# 도자기 공방 홈페이지 이슈 티켓 백로그

- 문서 버전: v1.0
- 작성일: 2026-04-24
- 상태: Draft
- 기준 문서
  - `pottery_technical_spec.md`
  - `pottery_execution_plan.md`

## 1. 문서 목적

이 문서는 실행계획을 실제 개발 티켓 단위로 쪼갠 백로그 문서다.
각 티켓은 우선순위, 선행조건, 완료 기준을 포함한다.

권장 사용 방식:

- 이 문서를 GitHub Issues, Notion Task DB, Linear 등으로 옮겨 사용
- 티켓 ID는 그대로 유지
- `Must` 범위를 먼저 끝내고 `Should`, `Could`는 여유 시 반영

## 2. 우선순위 기준

- `P0`: 오픈에 필수, 없으면 핵심 흐름 불가
- `P1`: 오픈 품질에 중요, 없으면 운영 부담 큼
- `P2`: 오픈 이후 개선 가능

## 3. 공통 Definition of Done

모든 티켓은 아래 기준을 기본으로 충족해야 완료로 본다.

- 로컬에서 동작 확인
- 타입 에러 없음
- lint 통과
- 에러 상태 처리 포함
- 모바일 화면 확인
- 관련 문서 또는 주석 업데이트
